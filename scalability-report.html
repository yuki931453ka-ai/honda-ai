Mac mini（自宅サーバー）

項目評価詳細
処理能力十分Flask + Gunicorn は軽量。同時50接続でも問題なし
ネットワーク課題大自宅回線のアップロード速度、固定IP不在、ポート開放のセキュリティリスク
可用性課題大停電・回線障害で全ユーザーがアクセス不能。SLA保証なし
SSL/TLS追加作業Let's Encrypt + Nginx リバースプロキシが必要
コスト電気代のみ月額 ~¥1,000（24時間稼働時）
結論: Mac mini は開発・テスト環境としては最適。
本番運用には Cloudflare Tunnel を使えば固定IP不要・SSL自動で自宅サーバーを公開可能（無料）。
ただし可用性のリスクは残るため、1,000人規模では Vercel + Supabase をメインに据えるのが安全。

7. 段階的スケールアップの手順

Phase 1: 最小限のデプロイ（〜50人）

目標: まず動くものを公開して、フィードバックを集める
変更点:
  • Vercel にデプロイ（vercel.json + requirements.txt 追加）
  • Anthropic API キーを Vercel 環境変数に設定
  • フィードバックは /tmp に書き込み（揮発性だが最小工数）
  • Haiku 4.5 に切り替えてコスト抑制
月額: Vercel Free + API ≈ ¥3,000〜¥5,000

Phase 2: データ永続化（〜200人）

目標: フィードバックと会話履歴を永続化
変更点:
  • Supabase にフィードバックテーブルを作成（JSONL → PostgreSQL）
  • 会話履歴を Supabase に保存（localStorage からの移行）
  • プロンプトキャッシュを有効化（コスト70%削減）
  • ストリーミング応答の実装（SSE）
月額: Vercel Free/Pro + Supabase Free + API ≈ ¥8,000〜¥20,000

Phase 3: 本格運用（〜1,000人）

目標: 安定運用と収益化
変更点:
  • Vercel Pro 移行（関数タイムアウト60秒）
  • Supabase Auth でユーザー認証（名前 → メール/SNSログイン）
  • 利用量制限の実装（1日の往復数上限 or サブスク課金）
  • 会話履歴の要約圧縮（長期会話のコスト抑制）
  • フィードバック分析ダッシュボード
  • ペルソナ自動改修パイプライン（将来）
月額: Vercel Pro ¥3,000 + Supabase Free ¥0 + API ¥18,000〜¥142,000
収益: ¥500/月 × 200人課金 = ¥100,000（黒字化ライン）
